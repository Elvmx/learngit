master 分支是一个主干分支，与线上版本保持一致，一般不在这里做开发。
dev 分支是一个开发分支，大家都往这个分支贡献代码，等准备发布的时候，再将 dev 合并到 master，然后使用 master 发布。
	两种情况：

		1. 大家都用 dev 分支。
		2. 大家基于 dev 分支，创建自己的分支，然后合并到dev
